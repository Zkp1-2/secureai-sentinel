# MITRE ATT&CK Initial Access
Attackers often gain initial access through public-facing applications, phishing, exposed services, or valid accounts. Defensive priorities include reducing external exposure, patching known vulnerabilities, and monitoring anomalous login patterns.

# MITRE ATT&CK Credential Access
Credential access includes stealing tokens, dumping credentials, searching configuration files, reading logs, and harvesting browser/session data. Defensive priorities include secret rotation, least privilege, credential scanning, and strong authentication.

# MITRE ATT&CK Privilege Escalation
Privilege escalation occurs when attackers move from low-privilege access to administrative control. Broken access control, misconfigured roles, and exposed admin functions can contribute to this behavior.

# MITRE ATT&CK Defense Evasion
Attackers may hide activity by disabling logs, using legitimate tools, or blending with normal traffic. Monitoring and tamper-resistant logging improve detection.

# MITRE ATT&CK Lateral Movement
After compromise, attackers may reuse credentials, tokens, or sessions to move to other systems. Network segmentation, least privilege, and credential rotation reduce blast radius.
